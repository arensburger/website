Leaning to use Command Lines

To communicate with the lab computer, you'll need to use command lines, most of the power of this operating system comes from learning to issue command via the command line. Why? Imagine you want to count the number of times the human genome has the sequence AAATCGC. If you try to open a 3,000,000,000 letter long document in Microsoft Word you'll likely crash your computer. However, issuing the command line below will give you the answer quickly even on a typical laptop computer (provided it runs Linux).

grep -o "AAATCGC" genome_file_name | wc -l

The structure of the command above will take a bit of work to understand, but hopefully you can see how useful command lines can be for bioinformatic work. Here's a basic tutorial on using a command lines in Ubuntu. It's probably best to work through this tutorial once you have a way to access a Linux based machine.

Which language should I learn?

This is nearly always the first question students ask when they think about learning to program. There’s no one answer, it depends on what you want to achieve. Think of programming languages as a set of tools in a toolbox; all toolboxes can solve basic problems, but some toolboxes are better suited to certain tasks than others. Some programming languages are better suited to solve certain types of computational problems than others. When it comes to bioinformatics, there are a number of programming languages that are commonly used. Three of the most commonly used are listed below along with their strengths and weaknesses:

R
strengths
  • Very popular in many different fields (not just bioinformatic)
  • Strong bioinformatic community
  • Many existing programs that can be combined to solve specific problem
weaknesses
  • Takes a statistical approach to problems, which is not always well suited for very large bioinformatic data sets
  • May take the most time to “get the hang of” of all the languages listed here
Python
strengths
  • Currently the most popular introductory programming language
  • Strong bioinformatic community
weaknesses
  • This language becomes less useful when used for more computationally demanding problem
Perl
strengths
  • The best language for manipulation of text (e.g. genome sequences)
  • Strong bioinformatic community
  • Many existing programs that can be combined to solve specific problem
weaknesses
  • This is not currently a popular language, so not as much demand for it
There are many, many, other languages (Java, C, etc.). Once you’ve mastered one programming language it becomes much easier to pick up other ones, most bioinformaticians end up using multiple languages even they have one preferred one.

I’ve picked a language, now what?

Now it’s time learn. There are as many ways to learn how to program, below are some resources but there are many, many others. Pick the one that work best for you.
  • Coursera, Programming for Everybody Four week online course that “aims to teach everyone the basics of programming computers using Python”
  • edX, Data Science: R Basics Eight week online course that will “help you master the basics of R”.
  • O’Reilly, Beginning Perl for bioinformatics Book that introduces both Perl programming and bioinformatic skills together. An online version of this book is available through the CPP library.
